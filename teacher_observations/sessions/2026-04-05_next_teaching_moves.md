# Next Teaching Moves

## Immediate Plan

- Start with one function only: `normalize_username`.
- Break the solution into the smallest possible Python steps.
- Teach exact syntax, not just concepts.
- Make the student write code line by line and explain each line.
- Update the observation files after each meaningful student reply so the plan stays current.
- Use the student's inline comments as part of the teaching review.

## What To Watch For Next

- Whether the student knows when to use `=` versus `==`
- Whether the student knows how to call methods such as `.strip()` and `.lower()`
- Whether the student can use intermediate variables
- Whether the student can read a failing test and connect it to their code
- Whether the student can distinguish data transformation from control flow
- Whether the student introduces extra code that is syntactically valid but logically unrelated
- Whether the student can turn a comment into one correct Python line
- Whether the student understands the difference between:
  - defining a function
  - calling a function
  - returning a value
  - printing a value
- Whether the student remembers Python-specific syntax when they already know the logic from another language
- Whether the student uses `#` comments instead of stray triple-quoted strings

## Success Criteria For The Next Session

- The student writes one complete working Python function with minimal help.
- The student can explain what each line does.
- The student can run tests and interpret at least one failure message.
- The student learns the specific difference between detecting a space and removing surrounding whitespace.
- The student reads one failing assertion and uses it to decide the next code change.
- The student can correctly predict when a script will print output and when it will not.
- The student can fix one syntax error from a traceback without guessing blindly.

## Ongoing Process Adjustments

- Keep a rolling teaching hypothesis instead of treating the first assessment as final.
- Revise notes when the student shows stronger or weaker understanding than previously recorded.
- Keep the repo tidy by removing artifacts that no longer help the learning process.
- Avoid cleaning up student attempts prematurely when they still provide useful evidence.
- Once syntax blockers are cleared, shift teaching attention to runtime behavior and data-shape assumptions.
