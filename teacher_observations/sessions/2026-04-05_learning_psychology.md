# Learning Psychology Notes

Date: 2026-04-05

## Student-Reported Tendencies

- The student self-reports some laziness.
- The student wants the teacher to notice this and plan around it.
- The student does not want immediate rescue through links or answers before making a serious attempt.
- The student appears to benefit from explicit next-step direction instead of an open-ended task list.
- The student can now distinguish between code they can improve alone and code they still need explained.

## Teacher Interpretation

- This suggests the student already knows that passive consumption is a risk.
- The student likely benefits from external structure, deadlines at the task level, and visible accountability.
- The teaching plan should aim to convert intention into repeated action.

## Strategic Teaching Response

- Give small, concrete tasks instead of broad assignments.
- Ask for an attempt before giving a link or direct fix.
- Use the student's own code and comments as the main teaching surface.
- Prefer short feedback loops:
  - attempt
  - run tests
  - review
  - retry
- Give targeted reading only when the gap is specific enough to matter.
- When momentum dips, reduce ambiguity by assigning exactly one next move.
- Let the student self-correct the parts they already recognize, then explain only the remaining unclear parts.
- When explaining a fix, include the exact rule and the reason behind it; this style appears to help the student internalize the concept.

## Risks To Watch

- Explaining instead of coding
- Wanting the answer before a real attempt
- Losing momentum when syntax gets frustrating
- Confusing "I understand it conceptually" with "I can do it on my own"

## Planning Implication

- Lessons should be narrow and action-heavy.
- Progress should be measured by completed code attempts, not by recognition alone.
- Reading assignments should be short, specific, and tied to a problem the student has already felt.
